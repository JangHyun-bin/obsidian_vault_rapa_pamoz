# 0. 본 문서의 배경

본 사업의 측위·AR 길안내 기능은 레몬케어 앱에 내부 SDK로 통합되는 모델로 제안되었습니다. 본 문서는 그 SDK 통합 모델을 구체화하기 위한 사전 확인 사항을 정리합니다.

## 0.1 사전 합의가 필요한 이유

* SDK 통합은 두 회사의 기술 스택·정책이 만나는 지점 — 사전 정렬 없이 진행 시 통합 단계에서 의존성 충돌·권한 충돌·진입점 재설계 발생 가능
* RAPA 1차년도 일정이 7개월로 압축되어 있어 재작업 부담 회피 필수
* 레몬케어 앱은 1,300만 가입자를 보유한 운영 앱 — SDK 통합으로 인한 안정성 영향 사전 검증 필요
* 일부 항목(인증·Rate Limit·분석 도구 정책)은 개발 시작 전 결정 안 되면 SDK 코드 자체를 못 씀

# 5. 권한 처리 ★★★

*본 사업 SDK가 필요로 하는 권한들이* **레몬 앱이 이미 받은 권한과 어떻게 정렬되는지** *사전 합의. 권한 요청 모달이 충돌하면 사용자 마찰 발생.*

## 5.1 본 사업 SDK가 요구하는 권한

| **권한** | **용도** | **필수성** |
| --- | --- | --- |
| **위치 (Location)** | 실내 측위 (BLE 비콘·지자기·UWB 결합). iOS는 BLE 백그라운드 스캔에도 필수 | ★ 필수 |
| **블루투스 (Bluetooth)** | BLE 비콘 RSSI 측정, BLE 5.1 AoA·UWB Nearby Interaction | ★ 필수 |
